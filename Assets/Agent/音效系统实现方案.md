# 任务：在 Audio 目录下实现一套简易 Unity 音效系统

## 任务目标

请在当前 Unity 项目的 `Audio` 目录中实现一套简易但结构清晰的音频系统，用于统一管理音效与 BGM 播放。

这套系统需要支持通过字符串 `Name` 查找音频配置并播放音频，同时能够管理运行中的音频状态、自动回收非循环音效、根据条件停止循环音效，并支持 BGM 的双 AudioSource 淡入淡出切换。

请优先保证代码清晰、稳定、易扩展、易在 Inspector 中配置。不要引入过度复杂的架构，不要依赖 Addressables 或额外资源加载系统，除非项目中已经存在明确可用的相关框架。

如果项目中已经安装 DOTween，可以使用 DOTween 实现 BGM 淡入淡出。如果项目中没有 DOTween，请使用 Coroutine 实现等价的淡入淡出逻辑。

---

## 一、需要实现的核心脚本

请至少实现以下四个核心类：

```csharp
AudioClipSO
AudioLibrarySO
AudioRuntime
AudioPlayer
```

可以根据实现需要添加辅助类、枚举或工具方法，但不要让系统变得过度复杂。

## 二、AudioClipSO：单个音频封装 ScriptableObject

请创建一个用于封装单个音频资源的 ScriptableObject，建议命名为：

```csharp
AudioClipSO
```

它只负责保存单个音频的静态配置，不负责播放逻辑。

### 必须包含字段

```csharp
[SerializeField] private string audioName;
[SerializeField] private AudioClip clip;
```

### 建议额外包含字段

请根据合理性添加以下配置字段：

```csharp
[SerializeField, Range(0f, 1f)] private float volume = 1f;
[SerializeField] private Vector2 randomPitchRange = new Vector2(1f, 1f);
[SerializeField] private bool defaultLoop = false;
[SerializeField] private float spatialBlend = 0f;
[SerializeField] private int priority = 128;
[SerializeField] private float defaultFadeDuration = 0.5f;
```

### 字段含义

`audioName` 是外部调用播放音频时使用的字符串 key。

`clip` 是实际播放的 Unity `AudioClip`。

`volume` 是该音频自身的默认音量倍率。

`randomPitchRange` 用于控制每次播放时的随机 pitch 范围。

`defaultLoop` 表示该音频配置上的默认循环倾向，但实际是否循环播放应由播放函数决定。

`spatialBlend` 用于控制 2D 或 3D 音效混合。

`priority` 对应 Unity AudioSource 的优先级。

`defaultFadeDuration` 可作为该音频默认淡入淡出时间。

### 实现要求

对外提供只读属性，例如：

```csharp
public string AudioName => audioName;
public AudioClip Clip => clip;
public float Volume => volume;
public Vector2 RandomPitchRange => randomPitchRange;
public bool DefaultLoop => defaultLoop;
public float SpatialBlend => spatialBlend;
public int Priority => priority;
public float DefaultFadeDuration => defaultFadeDuration;
```

播放时如果 `clip` 为空，需要输出明确 Warning。

如果 `randomPitchRange.x > randomPitchRange.y`，运行时需要自动修正，或者输出 Warning 后使用安全默认值。

## 三、AudioLibrarySO：音频库 ScriptableObject

请创建一个音频库 ScriptableObject，建议命名为：

```csharp
AudioLibrarySO
```

它负责保存多个 `AudioClipSO`，并在运行时根据 `AudioClipSO.AudioName` 建立字典，方便通过字符串 key 查询音频配置。

### 必须包含字段

```csharp
[SerializeField] private List<AudioClipSO> audioClips;
```

### 必须实现的运行时字典

```csharp
private Dictionary<string, AudioClipSO> audioDict;
```

### 必须提供的方法

```csharp
public void Init();

public bool TryGetAudio(string audioName, out AudioClipSO audio);

public AudioClipSO GetAudio(string audioName);
```

### 字典构建要求

`Init()` 可以被重复调用，但每次调用都应当重新构建字典，不能重复累积旧数据。

`audioClips` 中的空引用需要跳过，并输出 Warning。

`AudioClipSO.AudioName` 为空或 null 时需要跳过，并输出 Warning。

如果出现重复 `AudioName`，不应导致程序崩溃。默认保留第一个音频配置，跳过后续重复项，并输出 Warning。

查询不存在的音频时，返回 null 或 false，不要抛异常。

### 静态入口建议

音频系统需要提供方便的静态播放入口。

推荐由 `AudioPlayer` 持有 `AudioLibrarySO`，并通过 `AudioPlayer.PlayXXX(...)` 提供静态入口。

不推荐让 `AudioLibrarySO` 自己承担过多运行时职责。

## 四、AudioRuntime：运行中音频状态类

请创建一个普通 C# 类，建议命名为：

```csharp
AudioRuntime
```

它不需要继承 `MonoBehaviour`，用于表示一次音频播放实例的运行状态。

### 建议包含属性

```csharp
public int RuntimeId { get; }
public string AudioName { get; }
public AudioClipSO AudioData { get; }
public AudioSource Source { get; }
public bool IsLoop { get; }
public bool IsBGM { get; }
public bool IsReleased { get; private set; }
public bool IsStopping { get; private set; }
public float StartTime { get; }
public Func<bool> StopCondition { get; }
```

可以根据实现需要额外添加字段，例如：

```csharp
public float VolumeScale { get; }
public float FadeDuration { get; }
```

### 必须实现的功能

记录本次播放使用的 `AudioClipSO`。

记录本次播放使用的 `AudioSource`。

记录是否为 Loop 音频。

记录是否为 BGM。

记录播放开始时间。

支持条件停止判断。

支持手动停止，例如：

```csharp
public void Stop(float fadeDuration = 0f);
```

### 生命周期要求

非 Loop 音频播放完毕后，应由 `AudioPlayer` 自动检测并释放。

Loop 音频如果传入了 `Func<bool> stopCondition`，则当 `stopCondition.Invoke()` 返回 `true` 时，自动停止并释放。

`AudioRuntime` 不直接销毁 `AudioSource`，只记录状态并配合 `AudioPlayer` 释放资源。

Runtime 需要避免被重复停止或重复释放。

如果使用 DOTween，需要在停止或释放时安全 Kill 对应 Tween，避免 Tween 残留。

## 五、AudioPlayer：音频播放器 MonoBehaviour

请创建一个 MonoBehaviour，建议命名为：

```csharp
AudioPlayer
```

它是整个音频系统的运行核心。

### AudioPlayer 负责的内容

初始化音频库。

管理音效 AudioSource 池。

管理 BGM 双 AudioSource。

通过 string name 播放音频。

创建并维护 `AudioRuntime`。

自动回收播放结束的 Runtime。

处理 Loop 音频的条件停止。

处理 BGM 的淡入淡出切换。

---

## 六、AudioPlayer Inspector 配置

`AudioPlayer` 至少应当包含以下可配置字段：

```csharp
[SerializeField] private AudioLibrarySO audioLibrary;

[SerializeField] private int sfxSourcePoolSize = 16;
[SerializeField] private AudioSource sfxSourcePrefab;

[SerializeField] private AudioSource bgmSourceA;
[SerializeField] private AudioSource bgmSourceB;

[SerializeField] private float defaultBGMFadeDuration = 1f;
[SerializeField] private bool dontDestroyOnLoad = true;
```

可以根据需要添加其他配置字段，例如：

```csharp
[SerializeField] private bool logWarnings = true;
[SerializeField] private bool allowStealOldestSource = false;
```

### 初始化要求

`AudioPlayer` 可以实现单例，但必须处理重复实例问题，避免多个 `AudioPlayer` 同时工作。

`Awake()` 中需要初始化单例、音频库和 SFX Source 池。

如果 `dontDestroyOnLoad` 为 true，则播放器对象需要跨场景保留。

如果没有配置 `sfxSourcePrefab`，可以在运行时自动创建空 GameObject 并挂载 `AudioSource`。

SFX Source 池在 `Awake()` 中初始化。

BGM Source 必须独立于 SFX Source 池。

BGM 使用两个独立 `AudioSource` 交替播放。

## 七、AudioPlayer 静态播放接口

请提供静态方法作为外部主要调用入口。

---

### 1. 单次播放函数

用于播放一次非 Loop 音效。

建议接口：

```csharp
public static AudioRuntime PlayOneShot(string audioName, float volumeScale = 1f);
```

### 行为要求

通过 `audioName` 从 `AudioLibrarySO` 查找对应音频。

找不到时输出 Warning 并返回 null。

如果找到的 `AudioClipSO` 中 `clip` 为空，输出 Warning 并返回 null。

从 SFX Source 池中获取空闲 Source。

设置 Source 的 clip、volume、pitch、loop、spatialBlend、priority。

以非 Loop 方式播放。

创建一个 `AudioRuntime`。

将 Runtime 加入运行中列表。

播放结束后自动释放 Source，并从运行中列表移除。

---

### 2. 条件播放函数

用于播放一个 Loop 音频，直到条件触发后自动停止。

建议接口：

```csharp
public static AudioRuntime PlayLoopUntil(
    string audioName,
    Func<bool> stopCondition,
    float volumeScale = 1f
);
```

### 条件语义

`stopCondition == null` 时，默认不会自动停止，只能手动停止。

`stopCondition.Invoke()` 返回 `false` 时继续播放。

`stopCondition.Invoke()` 返回 `true` 时停止播放，并释放 Runtime。

### 行为要求

通过 `audioName` 从 `AudioLibrarySO` 查找对应音频。

找不到时输出 Warning 并返回 null。

如果找到的 `AudioClipSO` 中 `clip` 为空，输出 Warning 并返回 null。

从 SFX Source 池中获取空闲 Source。

以 Loop 方式播放。

创建 Runtime 并加入运行中列表。

在 `AudioPlayer.Update()` 中每帧检查停止条件。

如果条件函数执行时抛出异常，需要捕获异常，输出 Error，并停止该音频，避免异常刷屏。

---

### 3. BGM 播放函数

用于播放或切换 BGM。

建议接口：

```csharp
public static void PlayBGM(string audioName, float fadeDuration = -1f);
```

### 行为要求

通过 `audioName` 从 `AudioLibrarySO` 查找对应音频。

找不到时输出 Warning 并 return。

如果找到的 `AudioClipSO` 中 `clip` 为空，输出 Warning 并 return。

使用两个独立 BGM Source 交替播放。

当前 Source 淡出，新 Source 淡入。

`fadeDuration < 0` 时使用 `defaultBGMFadeDuration`。

BGM 不占用 SFX Source 池。

BGM 不参与普通 SFX Source 池回收逻辑，或者通过 `IsBGM` 区分处理。

如果请求播放的 BGM 与当前正在播放的 BGM 相同，默认不重新播放，直接 return。

BGM 的最终音量应当受到 `AudioClipSO.Volume` 和传入参数影响。

切换 BGM 时，旧 Source 淡出完成后停止并清空 clip。

---

## 八、Source 池规则

SFX Source 池需要满足以下规则：

播放音效时优先寻找未播放中的 Source。

如果 Source 池已满，可以采用以下策略之一：

1. 忽略本次播放并输出 Warning。
2. 抢占最早开始播放的非 Loop 音效。

默认推荐采用策略 1，逻辑更安全。

### 释放 Source 时需要重置状态

释放 SFX Source 时需要执行以下操作：

```csharp
source.Stop();
source.clip = null;
source.loop = false;
source.pitch = 1f;
source.volume = 1f;
source.spatialBlend = 0f;
source.priority = 128;
```

同时需要清除该 Source 与 Runtime 的关联。

## 九、运行时列表管理

`AudioPlayer` 中需要维护运行中音频列表，例如：

```csharp
private readonly List<AudioRuntime> runningAudios = new();
```

### 管理要求

所有通过播放器创建的 Runtime 都需要加入列表。

`Update()` 中需要检查：

1. 非 Loop 音频是否播放完毕。
2. Loop 音频的停止条件是否触发。
3. Runtime 是否已经释放。
4. Source 是否仍然有效。

遍历列表时注意不要在 `foreach` 中直接删除元素。

已释放 Runtime 要从列表中移除。

### 可选停止接口

可以额外提供以下静态停止函数：

```csharp
public static void Stop(AudioRuntime runtime, float fadeDuration = 0f);

public static void StopByName(string audioName, float fadeDuration = 0f);

public static void StopAllSFX(float fadeDuration = 0f);
```

如果实现这些函数，需要保持行为清晰：

`Stop(AudioRuntime runtime)` 只停止指定 Runtime。

`StopByName(string audioName)` 停止所有匹配该音频名的非 BGM Runtime。

`StopAllSFX()` 停止所有非 BGM 音频，不影响 BGM。

## 十二、最低验收标准

完成后，应当能够在其他脚本中正常调用：

```csharp
AudioPlayer.PlayOneShot("ButtonClick");

AudioRuntime runtime = AudioPlayer.PlayLoopUntil(
    "WarningAlarm",
    () => playerHp > 50
);

AudioPlayer.PlayBGM("BattleTheme", 1.5f);
```

### 预期行为

`"ButtonClick"` 播放一次，播放结束后自动释放 Source。

`"WarningAlarm"` 循环播放，直到 `playerHp > 50` 返回 true 后停止并释放。

`"BattleTheme"` 作为 BGM 播放，如果当前已有 BGM，则旧 BGM 淡出，新 BGM 淡入。

所有运行中的非 BGM 音频都能在 `runningAudios` 中被记录，并在结束后清理。

BGM 使用单独 Source，不占用音效池。

## 十三、补充建议

请注意 `PlayLoopUntil` 的停止条件语义是：

```csharp
Func<bool> stopCondition
```

返回 `true` 代表停止播放。

返回 `false` 代表继续播放。

因此调用侧示例中的条件应当写成“什么时候停止”，而不是“什么时候继续”。

例如：

```csharp
AudioPlayer.PlayLoopUntil(
    "LowHpWarning",
    () => playerHp > safeHpValue
);
```

这表示低血量警报会一直播放，直到玩家血量恢复到安全值以上后停止。
