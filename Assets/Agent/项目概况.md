> 项目路径：`d:/创三 种田游戏/GameDevelopment_III`  
> 引擎版本：Unity 2D (URP 渲染管线)  
> 游戏类型：2D 像素风农场模拟经营  
> 开发状态：**开发中**（核心框架已建立，具备可玩原型）

---

## 一、游戏简介

这是一款 2D 俯视角像素风格的农场模拟经营游戏，核心玩法借鉴《星露谷物语》。玩家在游戏世界中经营农场，进行**耕作、种植、收获、钓鱼、砍树、采集**等日常活动，并可通过自己的**商店摊位**向 NPC 顾客出售商品，构建完整的经济循环。

## 五、核心系统详解

### 5.1 全局游戏状态 - `WorldState`

> 文件：`Assets/Scripts/Data/WorldState.cs`（860 行）

**设计模式：单例（Singleton）**。作为全局唯一的游戏状态管理器，承担以下职责：

- **地图数据管理**：四层架构存储
  1. **基础地块层**（`BasicCellData[]` 一维数组）—— 每个格子的 `GridType`（Water/Soil/Farmland/Grass）和 `Empty` 占用状态
  2. **稀疏扩展层**（`Dictionary<Vector3Int, DetailedCellData>`）—— 只存储有特殊状态或有实体的格子（含水、施肥、实体 ID 列表）
  3. **运行时实体层**（`Dictionary<int, IEntityRuntime>`）—— 实体 ID 到运行时对象的映射
  4. **显示层**（3 层 Tilemap: `MainTile` / `OverlapTile` / `UperTile`）—— 仅负责视觉瓦片渲染
- **实体管理**：注册/注销/查找/销毁实体
- **物品放置**：支持按物品 ID 放置实体和瓦片
- **交互系统**：工具交互（锄地、浇水、砍树、钓鱼、摇铃吸引顾客）
- **掉落物生成**：在世界坐标生成可拾取的掉落物

### 5.2 时间系统 - `TimeManager`

- **时间模型**：**季节（春/夏/秋/冬）+ 日期 + 小时 + 分钟**
- **更新层级**：
  - Tick 更新（每帧）
  - 分钟更新（每游戏分钟触发）
  - 日期更新（跨天触发，如作物跨天生长）
- **注册机制**：实体实现 `ITickUpdatable` / `IMinuteUpdatable` / `IDateUpdatable` 接口，自动接收周期性通知

### 5.3 物品系统

- **物品定义**：`ItemBase_SO`（ScriptableObject），包含数字 ID、字符 ID、名称、堆叠数、基础价格、星级、精灵图
- **特性（Feature）组合模式**：每个物品可挂载多个特性 ScriptableObject
  - `Feature_Seed` —— 种子：可播种，关联作物生长图块和产物
  - `Feature_Placement` —— 放置物：支持多格占据（如房屋、工厂），带放置虚影预览
  - `Feature_Tools` —— 工具：锄头/斧头/水壶/鱼竿/铃铛，定义工具类型和交互行为
  - `Feature_Fertilizer` —— 肥料：可施放到农田提升肥力
- **物品注册表**：`ItemRegistry` + `ItemDatabaseSO`，通过物品 ID 快速查询

### 5.4 背包系统

- **容器模型**：`ItemContainer`（逻辑容器）存储物品栈列表
- **UI 视图**：`BackpackContainer`（背包）/ `InventoryContainer`（库存）
- **控制器**：`SlotController`（单例）—— 管理拾取、放置、交换、堆叠等 UI 操作
- **快捷栏**：10 格热键栏，支持鼠标滚轮和数字键 1-0 切换

### 5.5 耕作系统

耕作流程如下：

1. **锄地**：在 `Soil` 格子使用锄头 → 创建 `Farmland_Entity` → 在 `OverlapTile` 层放置农田瓦片
2. **播种**：在 `Farmland` 上使用种子物品 → 创建 `Crops_Entity` → 在 `UperTile` 层放置种子瓦片
3. **浇水**：使用水壶 → `Farmland_Entity.Water()` → 设置 `WaterTimeLeft`（水分剩余时间）
4. **施肥**：使用肥料 → `Farmland_Entity.ApplyFertilizer()` → 提升肥力等级和持续时间
5. **生长**：每分钟 `OnMinuteUpdate()` → 水分充足时推进生长阶段 → 切换对应瓦片
6. **收获**：交互 → `TryHarvest()` → 生成掉落物 → 销毁作物实体

**关键实体**：
- `Farmland_Entity` —— 农田（含水量、肥力、关联作物）
- `Crops_Entity` —— 作物（生长阶段数组、生长进度、成熟收获）
- `Tree_Entity` —— 树木（可被斧头砍伐，产出木材）

### 5.6 实体系统

- **接口**：`IEntityRuntime`（基础实体接口）、`IInteractable`（可交互）、`IUpdate`（时间更新）、`ISaveableEntity`（可存档）
- **基类**：`EntityRuntimeBase`（实现 EntityId、PivotPos、EntityInit、Save/Load 骨架）
- **实体工厂**：`EntityRuntimeFactory` 按枚举类型创建实体实例
- **注册工具**：`RuntimeRegisterUtility` 自动将实体注册到 TimeManager 和 WorldState
- **具体实体**：
  - `Farmland_Entity` —— 农田
  - `Crops_Entity` —— 作物
  - `Tree_Entity` / `TreeInitializer` —— 树木
  - `PlayerStore_Entity` —— 玩家商店摊位
  - `NPCStore_Entity` —— NPC 商店
  - `Entity_Factory` —— 加工工厂
  - `DroppedItem` —— 掉落物（自动拾取）

### 5.7 钓鱼系统

- **核心类**：`FishingSystem`（单例）
- **流程**：等待（游戏分钟）→ 鱼咬钩（现实秒）→ 钓鱼小游戏 → 获得物品
- **战利品表**：`FishingLootTable_SO`，基于权重的随机抽取
- **状态**：`Wait → Bite → Game`
- **注意**：钓鱼小游戏 UI（`FishingGameView`）目前为空壳，尚未完成

### 5.8 交易与经济系统

**玩家购买**：
- `NPCStore_Entity` —— NPC 商店界面，玩家可从 NPC 处购买物品

**玩家出售（摆摊）**：
- `PlayerStore_Entity` —— 玩家放置的商店摊位（含货架 + 销售位 + 排队系统）
- `CustomerCreator` —— 顾客生成器（对象池 + 动画变体 + 生成冷却）
- `CustomerController` —— 顾客行为控制（被吸引 → 排队 → 购买 → 离开）
- `CustomerAttractSystem` —— 摇铃吸引顾客
- **顾客状态机**：`Idle → Attracting(排队) → Buying(购买) → Quit(离开)`
- **经济模型**：顾客具有"吸引态度"和"购买态度"，排队等待时间影响购买意愿

### 5.9 加工制造系统

- **核心类**：`Entity_Factory` —— 工厂实体（多格放置）
- **配方**：`Recipe_SO`（ScriptableObject）—— 输入原料 + 输出产物 + 加工时间
- **流程**：检查原料 → 开始生产 → 计时 → 产出到产物容器

### 5.10 玩家控制器

- **类**：`PlayerController`
- **移动**：四方向（上/下/左/右），Rigidbody2D，平滑方向过渡
- **交互**：`SimpleInteract`（普通交互，如对话/拾取）/ `ToolInteract`（工具使用，如锄地/浇水）
- **面向方向**：影响交互检测的偏移量
- **玩家状态机**：`Idle → BasicMove → Interact / UseTool / FishingWait / FishingBite / FishingGame`

### 5.11 状态机框架

- **核心**：`StateMachine<TContext>`（泛型）—— 支持 `ChangeState` / `PushState` / `PopState`
- **状态基类**：`State<TContext>` —— Enter/Exit/Pause/Resume/Update/HandleEvent
- **中央管理器**：`StateMachineBrain`（单例）—— 注册所有状态机并统一更新
- **事件系统**：`FsmEvent` —— 时间/移动/动画/交互/任务/属性等事件类型
- **动作系统**：`ActionDefinition_SO`（ScriptableObject）—— 动画定义（含时间线事件和判定范围），`ActionRuntime` —— 运行时执行

### 5.12 输入系统

- **基于 Unity Input System**
- **输入动作资源**：`InputActionMain` / `InputActionPlayer`
- **输入管理器**：`InputManager` —— 处理移动、交互、背包、暂停、复合操作
- **热键**：数字键 1-0 切换快捷栏，鼠标滚轮滚动

## 六、架构模式总结

| 模式 | 应用位置 |
|------|----------|
| **单例模式** | WorldState, TimeManager, SlotController, FishingSystem, StateMachineBrain, CustomerCreator, CustomerAttractSystem |
| **状态机模式** | 玩家行为、顾客 AI（`StateMachine<T>` 分层状态机） |
| **组件特性模式** | 物品系统的 Feature 组合（ScriptableObject 列表） |
| **接口驱动** | IEntityRuntime, IInteractable, IUpdate, ISaveableEntity, IHoldInteract, IHoldTick |
| **工厂模式** | EntityRuntimeFactory 创建实体实例 |
| **MVC 风格** | ItemContainer（模型）+ ContainerView（视图）+ SlotController（控制器） |
| **稀疏存储** | 地图数据 = 基础数组 + 稀疏字典 + 实体注册表 |
| **对象池** | CustomerCreator 对顾客进行对象池管理 |
| **观察者模式** | TimeManager 统一通知时间更新 |
| **数据驱动** | 物品、配方、瓦片、动画定义全部基于 ScriptableObject |
